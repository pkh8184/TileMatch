#### 1. 작업 확인

가장 마지막 Phase 파일부터 확인 (Phase_1, Phase_2, Phase_3, Phase_4, Phase_5, Phase_6, Phase_7)
전체 리팩터링 계획: docs/plans/2026-02-08-full-refactor.md
과거 작업은 Phase_Beginning1, Phase_Beginning2, Phase_Beginning3 파일 참고
작업 도중 필요에 따라서 RelatedPhases에 기록된 Phase를 확인

#### 2. 작업 기록

작업중이거나 이미 작업했던 Phase를 다시 작업하는 경우 해당 Phase의 md파일을 찾아 그 파일에 기록
새로운 Phase일 경우 "Phase_숫자_yyyy-mm-dd" 파일명으로 md파일을 만들어서 여기에 기록
작업 내역은 아래 형식으로 기록
Phase : 현재 페이즈
Status : 이 페이즈의 상태 요약
LastUpdate : 마지막 갱신 일자
Branch : 작업중인 branch
RecentWork : 가장 최근에 한 작업 요약
MainWork : 남은 작업 목록
ResolvedWork : 해결한 작업 목록
MainIssues : 발생한 문제 목록
ResolvedIssues : 해결한 문제 목록
RelatedPhases : 연관된 페이즈 목록
